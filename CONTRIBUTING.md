# Contributing to Axiom

Thank you for your interest in Axiom Bot! Please read this guide carefully before contributing.

## Important Notice

Axiom Bot is a **proprietary project** with restricted rights. Before contributing, please understand:

- All contributions become the property of NextGenXplorrers Team
- The project has a restrictive license (no commercial use, no modification for redistribution)
- We accept contributions but maintain full control over the codebase

## How You Can Help

### Reporting Bugs

1. Check existing issues to avoid duplicates
2. Use the bug report template
3. Include:
   - Clear description
   - Steps to reproduce
   - Expected vs actual behavior
   - Environment details (Node version, OS)
   - Screenshots if applicable

### Suggesting Features

1. Open a feature request issue
2. Describe the feature clearly
3. Explain the use case
4. Consider implementation complexity

### Improving Documentation

- Fix typos and grammar
- Improve clarity
- Add examples
- Update outdated information

## What We Accept

| Type | Accepted |
|------|----------|
| Bug fixes | Yes |
| Documentation | Yes |
| Feature suggestions | Yes |
| Security patches | Yes |
| Major rewrites | No |
| Forking for commercial use | No |

## Code Guidelines

If submitting code:

### Style Guide

```javascript
// Use ES Modules
import express from 'express';

// Use meaningful variable names
const discordClient = new Client();

// Comment complex logic
// Fetch user data and validate permissions
async function fetchUserData(userId) {
  // ...
}
```

### Commit Messages

```
type: brief description

- Detailed point 1
- Detailed point 2

Types: fix, feat, docs, style, refactor
```

### Pull Request Process

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Test thoroughly
5. Submit PR with clear description
6. Wait for review
